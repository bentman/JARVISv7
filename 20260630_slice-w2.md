## Slice W2 — Settings Grouping and Default Correction

Status: planned. This document is an implementation contract for a coding agent. It is not completion evidence.

---

## Purpose

Correct the Slice W landing so the starter environment file, backend settings defaults, Operator Settings grouping, and desktop static contracts describe the same configuration model.

This slice is intentionally narrow. It separates local llama.cpp runtime settings from local Ollama app settings, aligns safe backend defaults with the starter environment posture, and keeps desktop Settings rendering driven by backend metadata.

---

## Agent Contract

- Preserve supported environment names unless a tested compatibility path replaces them.
- Treat `.env.example` headings as the normal Operator grouping source.
- Keep local llama.cpp and local Ollama app controls separate.
- Keep llama.cpp as the default local LLM runtime posture.
- Keep Ollama available as a development fallback or app integration.
- Keep the default wake posture unchanged; optional wake-provider values are not default behavior.
- Reuse existing settings, API route, desktop static, npm desktop static, and local LLM setup tests first.
- Do not update changelog or inventory in this correction pass.

---

## Baseline Consumed

- `.env.example` defines starter grouping for App Defaults, Local LLM intent (llama.cpp), Use Local Ollama intent, Optional services, and Optional wake service.
- `backend/app/core/settings.py` owns repo-safe defaults and effective values.
- `backend/app/api/routes/config.py` owns Operator field metadata and section grouping.
- `desktop/src/components/settings-panel.js` renders Operator Settings from backend metadata.
- Existing settings, API, and desktop static tests are the preferred validation homes.

## Target Observable Behavior

Operator Settings should show distinct sections:

```text
App Defaults

Local LLM intent (llama.cpp)
  local runtime intent
    model policy and override
    advanced llama.cpp
    sidecar overrides

Use Ollama App intent
  Ollama app enablement
    Ollama app model
    context overrides
    advanced Ollama endpoint

Optional Services
  Redis
  SearXNG
  DDGS
  Tavily

Optional Wake
  optional wake-provider values only if exposed
```

Backend defaults should be safe when `.env` and `.env.example` are absent. Starter defaults and previously advertised safe path/runtime defaults should live in `Settings`, not only in `.env.example`.
