# ARM64 Agent Prompt: Slice U4 Simple Path

Goal: acquire and enable a Qualcomm AI Hub Whisper-Base QNN STT model through the repo.

Rules:
- Use only `backend/.venv`.
- Keep `onnxruntime>=1.24.4` and `onnxruntime-qnn>=2.3.0`.
- Do not add old QNN pins, `<1.25`, `--no-deps`, force reinstall, or uninstall `onnxruntime`.
- Do not test random public S3 model URLs.
- Do not update app STT selection until the new model verifies.
- If provisioning, export, verification, or live QNN fails: stop, Do not write changelog or inventory, report exact evidence.

Implementation:
1. Add Qualcomm AI Hub export dependencies through existing provisioning:
   - `qai-hub>=...`
   - `qai-hub-models[whisper-base]>=...`
   Use `>=` unless a tested constraint is required.

2. Add a new STT model entry in `config/models/stt.yaml`:
   - name: `whisper-base-qaihub-precompiled-qnn-onnx-snapdragon-x`
   - runtime: `qnn_whisper`
   - devices: `[qnn]`
   - source.type: `qai_hub_export`
   - module: `qai_hub_models.models.whisper_base.export`
   - target_runtime: `PRECOMPILED_QNN_ONNX`
   - device: current Qualcomm AI Hub-compatible Snapdragon X target
   - required files: `encoder.onnx`, `decoder.onnx`, `metadata.json`
   - local_path: `models/stt/whisper-base-qaihub-precompiled-qnn-onnx-snapdragon-x`

3. Extend `scripts/ensure_models.py` for `source.type=qai_hub_export`.
   - Call `qai_hub_models.models.whisper_base.export.export_model(...)`.
   - Download generated assets into the model `local_path`.
   - Keep it idempotent: if files exist and QNN verification passes, do not re-export.

4. Add one shared QNN STT artifact verification helper.
   It must:
   - find `encoder.onnx`
   - find `decoder.onnx`
   - create QNN sessions with CPU fallback disabled
   - require `QNNExecutionProvider` primary
   - return one clear pass/fail reason

5. Update `ensure_models.py --verify-only` so `qnn_whisper` readiness means QNN encoder/decoder session loadability, not file presence.

6. Only after the new model passes verification, update:
   - `backend/app/runtimes/stt/stt_runtime.py`
   - `backend/tests/runtime/voice/test_stt_live.py`
   to use `whisper-base-qaihub-precompiled-qnn-onnx-snapdragon-x`.

Validation:
1. `backend\.venv\Scripts\python scripts\provision.py install`
2. `backend\.venv\Scripts\python scripts\provision.py verify`
3. `backend\.venv\Scripts\python scripts\ensure_models.py --family stt --model whisper-base-qaihub-precompiled-qnn-onnx-snapdragon-x`
4. `backend\.venv\Scripts\python scripts\ensure_models.py --family stt --model whisper-base-qaihub-precompiled-qnn-onnx-snapdragon-x --verify-only`
5. `backend\.venv\Scripts\python scripts\validate_backend.py profile`
6. `$env:JARVISV7_LIVE_TESTS="1"; backend\.venv\Scripts\python -m pytest backend\tests\runtime\hardware\test_qnn_gate_live.py -q`
7. `$env:JARVISV7_LIVE_TESTS="1"; backend\.venv\Scripts\python -m pytest backend\tests\runtime\voice\test_stt_live.py -q -k qnn`
8. `backend\.venv\Scripts\python scripts\validate_backend.py unit`
9. `backend\.venv\Scripts\python scripts\validate_backend.py regression`

Success:
- `provision.py verify` passes.
- `profile` shows `ep:QNNExecutionProvider`.
- `ensure_models.py --verify-only` proves QNN encoder and decoder sessions.
- live QNN STT transcript test passes.
- unit and regression pass.

Failure:
- Stop.
- Do not write changelog or inventory.
- Report exact command, output excerpt, failing file/path/package, and whether failure is provisioning, export, QNN session load, or transcript.
