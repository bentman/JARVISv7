# 20260504-slice_g.md
## Slice G — Cross-Session Episodic Retrieval

**Date:** 2026-05-04
**Status:** Proposed
**Depends on:** Slice F closed and verified (SYSTEM_INVENTORY.md)

---

## Purpose

Working memory (C.3) gives the assistant continuity within a session. After a session ends, that context is gone. Group G closes the gap: turn artifacts and sessions written to `data/turns/` and `data/sessions/` become a queryable episodic store that session N can draw on when answering questions that implicitly reference prior interactions.

This is the difference between a single-session tool and a persistent local assistant. The durable memory authority already exists — turn artifacts have been written since C.3. Group G adds the read path, the write policy governing what enters the episodic index, and the retrieval injection into prompt assembly.

## Out of Scope

- Semantic memory (vector embeddings, similarity search) — future; `slices.md` explicitly excludes it from Group G
- Cross-session summarization beyond recency-based recall — excluded per `slices.md`
- LLM-driven autonomous memory decisions — no LLM decides what to write or retrieve; write policy is explicit code, retrieval is deterministic
- Group I agent roles (Curator, Learner) — these consume episodic memory later; Group G builds the substrate
- Group H local llama.cpp (H.1), QNN (H.2), DirectML (H.3)
- Tool behavior changes — `ToolRegistry` and `ToolExecutor` are not touched; retrieval is not a tool
- New packages, external services, or runtime families
- Redis as durable memory authority — Redis is an acceleration cache only; all durable reads fall back to disk artifacts when Redis is down

## Sub-Slice G.1 — Episodic Write + Retrieve

### Why Here

Turn artifacts have been written since C.3. The missing piece is an episodic index that makes prior artifacts queryable by content, and a write policy that governs what enters and stays in that index. Without G.1, every session starts from zero regardless of history.

### Goal

G.1 establishes the episodic substrate: a text or voice turn can be written into episodic storage after artifact persistence (when policy allows), and episodic facts can be retrieved directly (recency/keyword) for proof across sessions. G.1 does not inject retrieved context into prompts and does not populate `TurnArtifact.retrieved_memory_refs`; that ownership remains in G.2.

### Scope

**`backend/app/memory/episodic.py`** — `EpisodicMemory`:

- `EpisodicEntry` dataclass: `turn_id: str`, `session_id: str`, `session_started_at: str`, `transcript: str | None`, `response_text: str | None`, `tools_invoked: list[str]`, `written_at: str`.
- `write_entry(artifact: TurnArtifact, policy: WritePolicy) -> EpisodicEntry | None`:
  - Reads the write policy to decide whether to write this turn. Returns `None` if policy says skip (e.g., failed turns, empty transcript, short response below threshold).
  - Constructs an `EpisodicEntry` from the artifact fields.
  - Persists to `data/memory/episodic/<session_id>/<turn_id>.json` via an append-or-create pattern. Uses `storage`-compatible JSON serialization; does not invent a new persistence format.
  - Idempotent: writing the same `turn_id` twice produces the same file; no duplication.
- `retrieve_recent(n: int = 5, base_dir: Path | None = None) -> list[EpisodicEntry]`:
  - Reads up to `n` most recent `EpisodicEntry` records from `data/memory/episodic/` sorted by `written_at` descending.
  - Returns entries across sessions — this is the cross-session retrieval.
  - Fail-closed: returns `[]` on any I/O error; never raises.
- `retrieve_by_keyword(keyword: str, n: int = 5, base_dir: Path | None = None) -> list[EpisodicEntry]`:
  - Simple substring match on `transcript` and `response_text` fields (case-insensitive).
  - Returns up to `n` matches sorted by `written_at` descending.
  - Fail-closed: returns `[]` on any error.

**`backend/app/memory/retrieval.py`** — `RetrievalManager`:

- `RetrievedFact` dataclass: `turn_id: str`, `session_id: str`, `content: str`, `source_field: str` (e.g. `"response_text"`, `"transcript"`), `relevance_method: str` (e.g. `"keyword"`, `"recency"`).
- `retrieve(query: str | None, n: int = 3, cache_manager: CacheManager | None = None, episodic: EpisodicMemory | None = None) -> list[RetrievedFact]`:
  - If `query` is not None: keyword retrieval via `episodic.retrieve_by_keyword()`.
  - If `query` is None: recency retrieval via `episodic.retrieve_recent()`.
  - Returns `[]` when no entries match or episodic is unavailable.
  - G.3 extends this method to consult the cache first.

**`backend/app/memory/write_policy.py`** — extend `WritePolicy`:
- Add `write_to_episodic_memory: bool = True`
- Add `episodic_min_response_length: int = 10` — turns with response shorter than this are not written to the episodic store
- Add `episodic_skip_failed_turns: bool = True` — turns where `failure_reason` is set are not written
- Add `episodic_retention_sessions: int = 20` — maximum number of sessions whose entries are retained; oldest are pruned on write

Retention pruning method:
- Candidates are derived from `data/memory/episodic/<session_id>/` subdirectories, not a global index.
- Session ordering uses the earliest `written_at` ISO timestamp found in the session’s episodic entries, or falls back to the corresponding `SessionArtifact.started_at` value from `data/sessions/<session_id>/session.json`.
- If no readable timestamp exists for a session directory, that session is skipped during pruning — fail closed; do not delete data on ambiguous metadata.
- Pruning removes the oldest `data/memory/episodic/<session_id>/` directories only, up to the count needed to bring the total under `episodic_retention_sessions`.

**`TurnEngine._record_artifact()`** — extend to call `episodic.write_entry(artifact, write_policy)` after writing the turn artifact, when `self.episodic` is set. The episodic write is fire-and-forget: a write failure is logged but does not fail the turn.

G.1 engine wiring ownership:
- G.1 adds an optional `episodic: EpisodicMemory | None = None` parameter to the `TurnEngine` constructor. This is the minimal wiring needed to make post-turn episodic writes live.
- G.1 does **not** inject retrieved context into prompts and does **not** populate `TurnArtifact.retrieved_memory_refs`; both are G.2 scope.
- G.2 adds retrieval-before-prompt behavior and may introduce the `cache_manager` constructor parameter for retrieval use.
- G.2 owns `retrieved_context` prompt injection and population of `TurnArtifact.retrieved_memory_refs`.

## Sub-Slice G.2 — Retrieval Injected into Prompt Assembly

### Why Here

Retrieval exists (G.1) but the turn engine does not yet use it. G.2 wires `RetrievalManager.retrieve()` into the turn path so retrieved context shapes the LLM response, and records retrieval provenance in the turn artifact.

### Goal

A live turn in session N whose prompt assembly receives retrieved facts from session N-1 includes those facts in the LLM prompt and records their source in `TurnArtifact.retrieved_memory_refs`.

### Scope

**`backend/app/cognition/prompt_assembler.py`** — extend `assemble_prompt`:
- Add `retrieved_context: list[RetrievedFact] | None = None` keyword parameter.
- When `retrieved_context` is not empty, append a "Relevant prior context:" section to the prompt with one line per `RetrievedFact` formatted as `[{fact.session_id[:8]}/{fact.turn_id[:8]}] {fact.content}`.
- Existing callers that do not pass `retrieved_context` are unaffected (parameter defaults to `None`).

**`backend/app/conversation/engine.py`** — extend `_run_reasoning_path()`:

- Pass the result as `retrieved_context` to `assemble_prompt`.
- Populate `TurnArtifact.retrieved_memory_refs` with the `turn_id` strings from the retrieved facts.
- `TurnEngine` constructor gains two optional parameters: `episodic: EpisodicMemory | None = None` and `cache_manager: CacheManager | None = None`. Both default to `None`; when `None`, retrieval is skipped silently.

## Sub-Slice G.3 — Redis-Cached Retrieval

### Why Here

Retrieval reads from `data/memory/episodic/` which may contain many JSON files as history grows. G.3 accelerates repeated lookups using the E.3 Redis cache layer. Redis is an acceleration layer only; durable authority stays on disk.

### Goal

A retrieval lookup hits the Redis cache on the second call for the same query/session; Redis-stopped path still returns results from disk.

### Scope

**`backend/app/memory/retrieval.py`** — extend `RetrievalManager.retrieve()`:
- Construct a cache key via `cache_manager.make_key()` (or `keys.make_key(NS_RETRIEVAL, query_hash, str(n))`).
- On cache hit (`cache_manager.get(key)` returns a value): deserialize and return the cached `list[RetrievedFact]`.
- On cache miss: perform disk retrieval, serialize result, call `cache_manager.set(key, value, ttl=DEFAULT_RETRIEVAL_TTL)`.
- When `cache_manager` is `None` or `cache_manager.is_available()` is `False`: skip cache, go directly to disk retrieval. Never raises; fail-closed to disk.
- `query_hash`: stable 8-char hex of `hashlib.md5(query.encode(), usedforsecurity=False).hexdigest()[:8]` — deterministic, non-cryptographic.

**`backend/tests/unit/memory/test_retrieval.py`** — extend:
- `test_retrieve_uses_cache_on_second_call` (mock `CacheManager`)
- `test_retrieve_populates_cache_on_first_call` (mock `CacheManager`)
- `test_retrieve_falls_back_to_disk_when_cache_unavailable` (mock `CacheManager.is_available()` → `False`)
- `test_retrieve_falls_back_to_disk_when_cache_down` (mock `CacheManager.get()` → `None`)

**`backend/tests/runtime/services/test_retrieval_cached_live.py`** — `@pytest.mark.live @pytest.mark.requires_docker`, `skipif SKIP_UNLESS_LIVE`:
- `test_retrieval_cache_hit_returns_same_results_as_miss` — call `retrieve()` twice for the same query; confirm both return the same facts
- `test_retrieval_cache_miss_timing_slower_than_hit` — measure wall time; cache hit should be faster (soft assertion: log timing, do not fail if delta is under 10 ms)
- `test_retrieval_falls_back_to_disk_when_redis_stopped` — stop Redis, confirm `retrieve()` still returns results from disk; restart Redis after

### Risks

- **Redis key collision:** retrieval keys must be namespaced to `NS_RETRIEVAL` (from `backend/app/cache/keys.py`). Confirm the namespace constant exists before G.3 begins.

New turns written after a cache hit will not be reflected until TTL expires (`DEFAULT_RETRIEVAL_TTL = 300` seconds from E.3). This is acceptable for the assistant use case but should be documented in `retrieval.py`.
- **Stopping and restarting Redis in a live test:** the live fallback test stops Redis via `docker stop jarvisv7-redis` and restarts it after. If the test process is interrupted mid-test, Redis may remain stopped. Document the manual restart command in the test docstring: `docker start jarvisv7-redis`.

## Slice-Level Closeout Criteria

Slice G is complete when all of the following are true:

1. G-G1 through G-G5 gates verified before the relevant sub-slice begins.
2. `EpisodicMemory.write_entry()` is called after each turn and persists a JSON entry under `data/memory/episodic/`.
3. Write policy correctly skips failed turns, short responses, and honors `write_to_episodic_memory: False`.
4. `RetrievalManager.retrieve()` returns facts from a prior session's artifact on both host classes (live proof in `test_retrieval_live.py`).
5. Retrieved facts appear in the LLM prompt via `assemble_prompt` and are logged in `TurnArtifact.retrieved_memory_refs`.
6. Turn artifact for a retrieval-enabled turn shows non-empty `retrieved_memory_refs` (inspectable from JSON file).
7. Redis-cached retrieval returns the same results as disk retrieval for the same query (live proof in `test_retrieval_cached_live.py`).
8. Redis-stopped path returns results from disk without raising.
