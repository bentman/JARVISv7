# Slice U4 Summary: Side-by-Side QNN Whisper Completion

## Summary

Slice U4 completed on Windows ARM64.

The initial blocker was not the U4 artifact shape alone. QNN provider registration was working, but ONNX Runtime did not assign Qualcomm AI Hub `EPContext` nodes to QNN when sessions were created through the older provider-list path. The working correction was to create QNN sessions through ONNX Runtime's EP-device API:

```text
SessionOptions.add_provider_for_devices(...)
```

After that correction, both the existing catalog QNN Whisper artifact and the new `whisper-qualcomm-qnn` side-by-side artifact loaded with `QNNExecutionProvider` primary and CPU fallback disabled.

`CHANGE_LOG.md` and `SYSTEM_INVENTORY.md` were not edited.
