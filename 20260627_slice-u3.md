## Important EPContext Requirements

ONNX Runtime EPContext docs say QNN EP accepts EPContext nodes when:

- `source` is `QNN` or `QnnExecutionProvider`
- context binary reference is in `ep_cache_context`
- for file-path model loading, relative context paths resolve from the ONNX model folder

Relevant attributes to inspect:

```text
source
ep_cache_context
embed_mode
main_context
partition_name
ep_sdk_version
hardware_architecture
onnx_model_filename
```

Do not hand-edit model metadata unless the edit is documented, reproducible, and generated from source/artifact metadata. Do not commit local binary edits.
