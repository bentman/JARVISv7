The selected profile must compile into a high-authority system instruction plus generation defaults. It must produce observable behavior differences in local llama.cpp and local Ollama turns. It must stop relying on many weak enum labels as model-facing instructions.

## Research findings to guide implementation

Industry-standard prompt control uses high-authority instructions and role separation, not a large collection of weak metadata labels.

OpenAI text generation guidance:

- The `instructions` parameter gives high-level instructions for model behavior, including tone, goals, and examples.
- Instructions take priority over user input.
- Developer messages are application-level rules and business logic ahead of user messages.
- User messages are inputs behind developer messages.

Source:

- `https://developers.openai.com/api/docs/guides/text`

Anthropic prompting guidance:

- Role-setting belongs in the system prompt.
- Clear, explicit instructions affect behavior and tone.
- Examples should be relevant and diverse. Use a small set of high-signal examples when examples are needed.

Source:

- `https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/system-prompts`

llama.cpp server guidance:

- The server exposes OpenAI-compatible `/v1/chat/completions` using a `messages` payload.
- Z.5B must keep llama.cpp personality switching on the OpenAI-compatible chat-completions path.

Source:

- `https://raw.githubusercontent.com/ggml-org/llama.cpp/master/tools/server/README.md`

Ollama API guidance:

- `/api/chat` accepts a `messages` payload and is the dynamic runtime path for selected personality profiles.
- `/api/create` supports creating stable model variants and remains future work for installed persona variants. Do not activate model creation in this slice.

Source:

- `https://raw.githubusercontent.com/ollama/ollama/main/docs/api.md`

Cloud escalation boundary:

- OpenAI escalation must map the compiled profile system instruction to `instructions` or a developer message.
- Anthropic escalation must map the compiled profile system instruction to the provider `system` field.
- Cloud runtimes remain policy-gated placeholders in this slice. Do not enable cloud calls.

## Required target profile contract

Replace the active profile contract with this smaller model-facing shape:

```yaml
profile_id: concise
display_name: Quinn
description: Terse general assistant.
locale: en
system: >-
  You are Quinn, a concise general assistant. Give the answer first.
  Keep answers under 50 words unless the user asks for detail.
  Do not add reassurance, banter, or background unless necessary.
style:
  max_words_default: 50
  structure: Direct answer, then one caveat only when needed.
  do:
    - Lead with the answer.
    - Use short sentences.
    - Prefer concrete next actions.
  avoid:
    - Filler.
    - Compliments.
    - Long explanations.
examples:
  - user: I have 20 minutes before guests arrive and the kitchen is messy. What should I do first?
    assistant: Clear visible clutter, wipe counters, empty the sink, and take out trash. Skip deep cleaning; make the room look usable.
generation:
  temperature: 0.25
  top_p: 0.8
  top_k: 25
  repeat_penalty: 1.12
  max_tokens: 90
  stop:
    - "\nUser:"
    - "\nAssistant:"
enabled: true
```

Target fields:

- `profile_id`: required non-empty string, safe profile id.
- `display_name`: required non-empty string.
- `description`: required non-empty string for UI and API summary.
- `locale`: required non-empty string.
- `system`: required non-empty string and primary high-authority behavior instruction.
- `style.max_words_default`: required positive integer.
- `style.structure`: required non-empty string.
- `style.do`: required list of non-empty strings.
- `style.avoid`: required list of non-empty strings.
- `examples`: required list of example objects with `user` and `assistant` non-empty strings.
- `generation`: required mapping using the current supported runtime generation keys: `temperature`, `top_p`, `top_k`, `repeat_penalty`, `max_tokens`, and `stop`.
- `enabled`: required boolean.

Remove active config use of these old fields:

- `tone`
- `brevity`
- `formality`
- `system_prompt_addendum`
- `response_language`
- `system_prompt`
- `style_rules`
- `speech_rules`
- `example_messages`
- `identity_summary`
- `warmth`
- `assertiveness`
- `humor_policy`
- `response_style`
- `acknowledgment_style`
- `confirmation_style`
- `interruption_style`
- `voice_pacing`
- `voice_energy`

Keep prohibited authority fields rejected:

- `tool_permissions`
- `tool_policy`
- `routing_policy`
- `model_routing`
- `memory_policy`
- `memory_permissions`
- `safety_overrides`
