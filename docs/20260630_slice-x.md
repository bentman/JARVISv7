# 20260630_slice-x.md
## Slice X — Local LLM Tier Activation

Status: planned. This document is an implementation contract. It is not completion evidence.

---

## Purpose

Activate real local LLM model tiers through the existing repo-owned model-selection, setup, readiness, Operator, and llama.cpp runtime paths.

Existing flow to preserve:

```text
config/models/llm.yaml -> select_llm_model(...) -> resolve_llm_serve_profile(...) -> ensure_models.py -> llama.cpp sidecar -> local runtime/readiness
```

Slice X is complete only when application behavior improves end-to-end:

- new clones set up the selected local model through existing setup commands
- existing repos move from the legacy development model to the new defined model ladder without manual repair
- development mode preserves the current Qwen2.5 behavior
- production mode uses the new model ladder and is independent of the legacy development model artifact
- Operator and readiness surfaces show truthful selected mode, policy, role, and model state
- existing voice, text, local turn, setup, readiness, Operator, and desktop static behavior do not regress

---

## Implementation Discipline

KISS/YAGNI means no unnecessary architecture, no speculative feature work, and no duplicate workflows. It does not reduce the required Slice X function.

Do not add a parallel selector, new setup entry point, desktop-only model system, or committed model files.

Use existing repo-owned code. Change existing code where required to make the intended function work safely.

## Required Mode Gate

Add an explicit model mode gate (in `## App Defaults` for `LLM_MODEL_MODE={dev;prod}`).

Environment variable=value:

```text
LLM_MODEL_MODE=dev
```

Required surfaces:

- `.env.example` contains `LLM_MODEL_MODE=dev` in the Local LLM intent section.
- `backend/app/core/settings.py` defines `llm_model_mode` with default `dev`.
- `SETTING_ENV_CLASSIFICATION` classifies `LLM_MODEL_MODE` as `primary`.
- Blank `LLM_MODEL_MODE` resolves to `dev`.
- Valid values are exactly `dev` and `prod`.
- Invalid values fail closed with a visible configuration/readiness error.
- Operator Settings exposes `LLM_MODEL_MODE` as a primary Local LLM intent select.
- Readiness/status reports selected model mode.
- `scripts/ensure_models.py` and `scripts/bootstrap.py` use the selected model mode.

Selection precedence:

```text
1. Nonblank LLM_MODEL_ID selects that explicit model after validation.
2. LLM_MODEL_MODE=dev selects assistant-small-q4.
3. LLM_MODEL_MODE=prod uses the production model ladder and policy mapping.
```

Development mode behavior:

```text
mode: dev
selected model: assistant-small-q4
source model: existing Qwen2.5 0.5B GGUF entry
purpose: current fast development/default behavior
policy role: dev
```

Production mode behavior:

```text
mode: prod
selected model: resolved by LLM_MODEL_POLICY using the model ladder in this document
purpose: functional local assistant model tiering
policy role: diagnostic, portable, balanced, quality, or vision_preview
```

`assistant-small-q4` remains a development and explicit legacy compatibility model. It is not selected by production Auto, Portable, Balanced, Quality, or Vision Preview behavior.

Legacy model removal guarantee applies to production policy-selected operation. Development mode intentionally uses `assistant-small-q4` and visibly reports the missing development model artifact when that artifact is absent.

## Current Repo Baseline

`config/models/llm.yaml` already defines:

- `llm_selection.default_policy: auto`
- Operator env keys `LLM_MODEL_POLICY` and `LLM_MODEL_ID`
- roles `portable`, `balanced`, `quality`, `vision_preview`, and `diagnostic`
- hardware selectors for AMD64, ARM64, CUDA, AMD GPU, Intel GPU, Qualcomm Adreno OpenCL, Qualcomm NPU base, and Qualcomm QNN
- llama.cpp serve profiles under the current model entry

Current limitation:

```text

```

Desktop and Operator baseline:

```text

desktop/tests/static.test.mjs is the desktop-owned static test surface.
```

Slice X adds the mode gate, activates the production ladder, and updates the backend/desktop settings contracts using existing metadata-driven rendering.

## Defined Production Model Ladder

Implement these production model ids. The implementation agent does not choose the model family, tier, or policy target.

| Role | Model id | Model source target | Function |
|---|---|---|---|
| `diagnostic` | `assistant-qwen3-0p6b-q8-diagnostic` | Qwen3 0.6B GGUF, Q8-class quantization | Fast production smoke tests and lowest-cost production fallback |
| `portable` | `assistant-qwen3-4b-q4-portable` | Qwen3 4B GGUF, Q4_K_M-class quantization | Default production cross-host local assistant model |
| `balanced` | `assistant-qwen3-8b-q5-balanced` | Qwen3 8B GGUF, Q5_K_M-class quantization | Better production reasoning on proven AMD64 CUDA |
| `quality` | `assistant-qwen3-14b-q4-quality` | Qwen3 14B GGUF, Q4_K_M-class quantization | Explicit production quality tier on AMD64 CUDA |
| `vision_preview` | `assistant-qwen3-0p6b-q8-diagnostic` | diagnostic model with placeholder role status | No vision runtime claim |

Artifact metadata is required. Before adding each catalog entry, verify and record the exact source repo, filename, quantization, expected size, local path, license/provenance, and route support from the model source. Do not invent metadata.

Keep `default_model` aligned with development mode:

```text
default_model: assistant-small-q4
```

Production selection must not rely on `default_model`. Production selection must use `LLM_MODEL_MODE=prod` plus `LLM_MODEL_POLICY` and the production role mapping.

## Required Production Policy Mapping

Implement this production policy behavior exactly.

```yaml
llm_selection:
  default_policy: auto
  modes:
    dev:
      model: assistant-small-q4
      role: dev
      status: active-development-default
    prod:
      role_mapping: production
  roles:
    diagnostic:
      model: assistant-qwen3-0p6b-q8-diagnostic
      status: active
    portable:
      model: assistant-qwen3-4b-q4-portable
      status: active
    balanced:
      model: assistant-qwen3-8b-q5-balanced
      status: active
    quality:
      model: assistant-qwen3-14b-q4-quality
      status: active-amd64-cuda
    vision_preview:
      model: assistant-qwen3-0p6b-q8-diagnostic
      status: placeholder-no-vision-runtime
  policies:
    auto:
      windows_amd64_gpu_nvidia_cuda: balanced
      windows_amd64_cpu: portable
      windows_arm64_cpu: portable
      windows_amd64_gpu_amd: portable
      windows_amd64_gpu_intel: portable
      windows_arm64_gpu_qualcomm_adreno_opencl: portable
      windows_arm64_npu_qualcomm_base: portable
      windows_arm64_npu_qualcomm_qnn: portable
    portable:
      "*": portable
    balanced:
      windows_amd64_gpu_nvidia_cuda: balanced
      "*": portable
    quality:
      windows_amd64_gpu_nvidia_cuda: quality
      "*": portable
    diagnostic:
      "*": diagnostic
    vision_preview:
      "*": vision_preview
```

Readiness must make mode and degraded policy mapping visible. When `LLM_MODEL_MODE=prod` and Balanced or Quality policy resolves to Portable on non-CUDA hardware, readiness reports requested mode, requested policy, selected role, selected model, and reason. It must not present Balanced or Quality as active on unsupported hardware.

## Required Outcomes

### New Clone Setup

- A new clone with no local `.env` uses `LLM_MODEL_MODE=dev` and selects `assistant-small-q4`.
- A new clone with `LLM_MODEL_MODE=prod` uses the production model ladder.
- `scripts/bootstrap.py` remains the normal setup command.
- `scripts/ensure_models.py` fetches/verifies the selected model and current-host runtime sidecar for the selected mode.
- No normal setup path requires raw `LLAMA_CPP_MODEL_PATH` editing.

### Existing Repo Transformation

- Blank `LLM_MODEL_MODE` resolves to `dev`.
- Absent `LLM_MODEL_MODE` resolves to `dev`.
- Setting `LLM_MODEL_MODE=prod` activates the production model ladder.
- Blank `LLM_MODEL_ID` uses mode plus policy selection.
- Absent `LLM_MODEL_ID` uses mode plus policy selection.
- Explicit `LLM_MODEL_ID=assistant-small-q4` remains explicit legacy override behavior.
- Clearing the legacy override returns to mode plus policy selection.
- Existing legacy artifacts may remain on disk.
- Production policy-selected operation does not require legacy artifacts.

### Legacy Model Independence

- Prod Auto does not select `assistant-small-q4`.
- Prod Portable does not select `assistant-small-q4`.
- Prod Balanced does not select `assistant-small-q4`.
- Prod Quality does not select `assistant-small-q4`.
- Prod Vision Preview does not select `assistant-small-q4`.
- Production policy-selected operation succeeds with no dependency on the legacy model artifact.
- Development mode remains the explicit owner of `assistant-small-q4`.

### Operator and Readiness UX

- Operator selects model mode first.
- Operator selects model policy second.
- Raw model id/path/binary controls remain advanced.
- Readiness/status reports requested mode, requested policy, selected role, concrete model id, serve profile, accelerator/degraded state, and selection reason.
- Unsupported acceleration produces visible degraded/unavailable state.
- Ollama remains a separate fallback/development path, not the default policy-selected local LLM path.

### Holistic Application Compatibility

- Catalog loading still works.
- Model selection still resolves by mode, route, policy, override, and hardware selector.
- Serve-profile resolution still returns a valid llama.cpp sidecar profile.
- Setup still uses `scripts/bootstrap.py`.
- Model/runtime acquisition still uses `scripts/ensure_models.py`.
- Local llama.cpp startup still passes the selected model id into serve-profile resolution.
- Readiness/status still exposes selected model truth.
- Operator settings still expose mode and policy first and raw overrides as advanced controls.
- Desktop settings rendering remains metadata-driven.
- Text/local-turn routes still use the selected local LLM path.
- Voice routes remain functional.
- Unsupported hardware paths degrade explicitly instead of selecting unavailable artifacts silently.

A catalog-only change is not complete.
