## Slice X — Local LLM Tier Activation (take two)

Status: planned. This document is an implementation contract. It is not completion evidence.

Baseline: revert commit `18f769023aa566c3cdd5c291f5be88fc4c243c95`.

## Objective

Activate real local LLM tiers by extending existing repo mechanisms. The result must preserve current development behavior and add production behavior.

Required functional result:

- `LLM_MODEL_MODE=dev` keeps the current `assistant-small-q4` Qwen2.5 0.5B behavior.
- `LLM_MODEL_MODE=prod` uses a defined Qwen3 production ladder.
- New clone setup continues through `scripts/bootstrap.py` and `scripts/ensure_models.py`.
- Existing repos transition by env values, with `LLM_MODEL_ID` cleared for policy selection.
- Production policy-selected operation does not depend on `assistant-small-q4`.
- Operator settings, desktop settings, readiness, setup, local turn, and voice paths stay compatible.

Existing flow to preserve:

```text
config/models/llm.yaml
-> backend/app/models/llm_selection.py::select_llm_model(...)
-> backend/app/models/llm_profiles.py::resolve_llm_serve_profile(...)
-> scripts/ensure_models.py
-> backend/app/services/local_llm_startup.py
-> backend/app/runtimes/llm/local_runtime.py
-> readiness / Operator / desktop metadata
```

## Corrective rule

Take two must extend current repo patterns. Do not invent new shapes.

Use these existing surfaces:

- `config/models/llm.yaml` for catalog, `llm_selection`, roles, policies, model source metadata, routes, generation defaults, and serve profiles.
- `backend/app/core/settings.py` for env classification, helpers, defaults, and `Settings` fields.
- `backend/app/api/routes/config.py` for Operator metadata.
- `desktop/src/components/settings-panel.js` for metadata-driven rendering.
- `backend/app/models/llm_selection.py` for policy selection.
- `backend/app/models/llm_profiles.py` for serve-profile resolution.
- `scripts/ensure_models.py` for model and runtime acquisition/verification.
- Existing backend and desktop tests.

Do not add YAML anchors, aliases, generated catalog blocks, implicit inherited model entries, a new registry, a new setup workflow, a desktop model manager, a new runtime launcher, model artifacts, changelog entries, or inventory entries.

## Required additions and changes

### Env and settings

Add `LLM_MODEL_MODE=dev` to `.env.example` under `## Local LLM intent (llama.cpp)`, after `USE_LOCAL_MODEL=true` and before `LLM_MODEL_POLICY=auto`.

Add to `backend/app/core/settings.py`:

```text
SETTING_ENV_CLASSIFICATION["LLM_MODEL_MODE"] = "primary"
_env_choice(name, choices, default)
Settings.llm_model_mode defaulting to dev
```

Required behavior:

```text
missing LLM_MODEL_MODE -> dev
blank LLM_MODEL_MODE -> dev
LLM_MODEL_MODE=dev -> valid
LLM_MODEL_MODE=prod -> valid
all other values -> ValueError naming LLM_MODEL_MODE
```

### Operator metadata

Add this field in `backend/app/api/routes/config.py` after `USE_LOCAL_MODEL` and before `LLM_MODEL_POLICY`:

```python
OperatorFieldSpec(
    "LLM_MODEL_MODE",
    "Model mode.",
    options=("dev", "prod"),
    section="Local LLM intent (llama.cpp)",
)
```

Do not change `settings-panel.js` for this field. The existing metadata renderer must render it.

### LLM catalog

Keep:

```text
default_model: assistant-small-q4
```

Edit existing `llm_selection` to add mode support:

```yaml
operator_env:
  mode: LLM_MODEL_MODE
  policy: LLM_MODEL_POLICY
  model_override: LLM_MODEL_ID
modes:
  dev:
    model: assistant-small-q4
    role: dev
    status: active-development-default
  prod:
    role_mapping: production
```

Production roles:

```yaml
diagnostic: assistant-qwen3-0p6b-q8-diagnostic
portable: assistant-qwen3-4b-q4-portable
balanced: assistant-qwen3-8b-q5-balanced
quality: assistant-qwen3-14b-q4-quality
vision_preview: assistant-qwen3-0p6b-q8-diagnostic with placeholder-no-vision-runtime
```

Production policies:

```text
auto: AMD64 CUDA -> balanced; every other current selector -> portable
portable: all selectors -> portable
balanced: AMD64 CUDA -> balanced; all other selectors -> portable
quality: AMD64 CUDA -> quality; all other selectors -> portable
vision_preview: all selectors -> vision_preview
diagnostic: all selectors -> diagnostic
```

Add four explicit complete model entries under `models:`:

```text
assistant-qwen3-0p6b-q8-diagnostic
assistant-qwen3-4b-q4-portable
assistant-qwen3-8b-q5-balanced
assistant-qwen3-14b-q4-quality
```

Each entry must follow the existing `assistant-small-q4` shape exactly:

```text
family, runtime, format, quantization, local_path, source, routes, generation_defaults, serve_profiles.hardware_profiles
```

Each entry must declare every current `assistant-small-q4` hardware profile. Source repo id, filename, expected size, local path, and quantization must be verified before entry creation. No placeholder artifact metadata.

## Required code behavior

Selection precedence:

```text
1. Nonblank LLM_MODEL_ID explicit override after route validation.
2. LLM_MODEL_MODE=dev selects assistant-small-q4.
3. LLM_MODEL_MODE=prod selects through LLM_MODEL_POLICY and production role mapping.
```

Update in place:

```text
backend/app/models/llm_selection.py: add mode to LLMModelSelection and extend select_llm_model without adding a parallel selector.
backend/app/models/llm_profiles.py: add model_mode to LLMServeProfileResolution only; keep policy logic out of this file.
backend/app/services/local_llm_startup.py: propagate mode from selection to resolution to LlamaCppLLM.
backend/app/runtimes/llm/local_runtime.py: store model_mode metadata only.
backend/app/routing/runtime_selector.py: add model_mode to SelectionTrace.
backend/app/api/schemas/readiness.py and routes/readiness.py: expose model_mode through existing LLM readiness family response.
scripts/ensure_models.py: preserve existing CLI and include mode, policy, role, hardware_selector, and model in selected LLM output.
scripts/bootstrap.py: preserve existing setup orchestration and do not duplicate LLM selection reporting.
```

Required selection outcomes:

```text
dev on AMD64 CUDA -> assistant-small-q4, role dev
prod auto on AMD64 CUDA -> assistant-qwen3-8b-q5-balanced
prod auto on AMD64 CPU -> assistant-qwen3-4b-q4-portable
prod balanced on non-CUDA -> portable while reporting requested policy balanced
prod quality on non-CUDA -> portable while reporting requested policy quality
prod vision_preview -> assistant-qwen3-0p6b-q8-diagnostic with placeholder-no-vision-runtime
prod policies never select assistant-small-q4
```

Do not claim vision, AMD GPU, Intel GPU, Adreno, NPU, or QNN LLM acceleration.
