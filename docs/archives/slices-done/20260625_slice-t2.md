# 20260625_slice-t2.md
## Slice T2 - Resident Voice Corrective Activation and Proof

---

## Purpose

Slice T2 corrects the gap between the Slice T.1-T.8 scaffold and the actual resident voice outcome required by `20260625_slice-t.md`. The current codebase has resident audio, VAD, wake/PTT integration points, barge-in wiring, diagnostics, and desktop status UI, but the documented evidence shows live resident microphone, wake, barge-in, hands-free, and desktop operator proof were skipped. This slice activates the resident stream through an explicit lifecycle, makes status truth match active behavior, prevents live interruption monitoring from hanging normal TTS playback, implements real backend resident modes, proves live paths under explicit gates, and records corrective governance truth.

Done means JARVISv7 has an active, truthfully reported resident voice layer with live evidence for the host classes claimed. Unit/static scaffolding alone is not sufficient closeout evidence.

- `20260625_slice-t.md`: defines the original Slice T target and closeout criteria; consumed as the acceptance contract.
- `architect_slice-t9-t16.md`: independent review identifying the same scaffold-vs-active-runtime gap; consumed as corrective analysis.
