## Slice Z.4 — Desktop State Smoothing

Status: planned. This document is an implementation contract. It is not completion evidence.

## Purpose

Desktop state display should have two visible surfaces:

- `System State` for overall backend status.
- `Turn Status` for the current conversation turn phase.

Other panels remain diagnostics.

## Current code evidence

Current inspected files:

- `desktop/src/index.html`: header has `#startup-state`; Conversation header has `#turn-state`; controls have `#voice-status`; readiness uses `#readiness-panel`; debug details remain under Conversation.
- `desktop/src/main.js`: `setState`, `showError`, `renderReadiness`, `refreshSessionStatus`, `startDesktop`, `submitText`, and `invokeResidentPtt` currently write or drive visible state.
- `desktop/src/components/state-label.js`: labels already cover `IDLE`, `LISTENING`, `TRANSCRIBING`, `REASONING`, `ACTING`, `RESPONDING`, `SPEAKING`, `INTERRUPTED`, `RECOVERING`, `DEGRADED`, and `FAILED`.
- `desktop/src/components/readiness-panel.js`: readiness summary currently renders `Arch`, `Profile`, `LLM`, and `Status`.
- `desktop/tests/static.test.mjs`: static tests are the desktop rendering contract.

## Target display

Header row:

```text
[JARVISv7 title] [Turn Status card over Conversation] [System State card over Operator]
```

`System State` replaces the current `State` label. It is right-aligned and sized to the Operator column.

`Turn Status` is a two-row card:

```text
Turn Status
IDLE • LISTEN • TRANSCRIBE • REASON • ACT • RESPOND • SPEAK • INTERRUPT • RECOVER • FAILED
```

Label mapping:

| Backend state | Display label |
|---|---|
| `IDLE` | `IDLE` |
| `LISTENING` | `LISTEN` |
| `TRANSCRIBING` | `TRANSCRIBE` |
| `REASONING` | `REASON` |
| `ACTING` | `ACT` |
| `RESPONDING` | `RESPOND` |
| `SPEAKING` | `SPEAK` |
| `INTERRUPTED` | `INTERRUPT` |
| `RECOVERING` | `RECOVER` |
| `FAILED` | `FAILED` |

Inactive rail states are gray. Active `IDLE` glows with highlight color. Active normal turn states glow green. Active `INTERRUPT` and `RECOVER` glow orange. Active `FAILED` glows red.

## State source

Add the smallest desktop state coordinator, for example `desktop/src/components/desktop-state.js`.

The coordinator owns visible rendering for:

- `System State`
- `Turn Status`

Inputs:

- backend lifecycle from `startDesktop` and error handling
- readiness payload from `renderReadiness`
- session payload from `refreshSessionStatus`
- local pending text state from `submitText`
- local pending PTT state from `invokeResidentPtt`

System State should represent startup, ready, degraded, failed, or backend unavailable.

Turn Status should represent local pending turn state first, then `/session/status.state`, then `IDLE` when the active session is idle.

## Rendering changes

- Rename header `State` to `System State`.
- Right-align the System State label and displayed value.
- Size System State to the Operator column.
- Add the Turn Status card over the Conversation column.
- Remove visible `#turn-state` from the Conversation header after the Turn Status card exists.
- Remove the `Status` fact from Readiness summary.
- Keep readiness family rows, Degraded list detail, Wake diagnostics, Resident Voice diagnostics, Services diagnostics, and Conversation debug details.
- Keep `#voice-status` as neutral control helper text while Turn Status owns the visible phase display.

## Desktop contract update process

Update `desktop/tests/static.test.mjs` in the same implementation as the HTML/CSS/component changes. The test should assert:

- `System State` label exists.
- `Turn Status` card exists.
- rail labels include `IDLE`, `LISTEN`, `TRANSCRIBE`, `REASON`, `ACT`, `RESPOND`, `SPEAK`, `INTERRUPT`, `RECOVER`, and `FAILED`.

## Acceptance criteria

- Header shows right-aligned `System State` sized to the Operator column.
- Header shows `Turn Status` centered over the Conversation column.
- Turn Status rail renders the compact labels listed above.
- Active rail color uses idle highlight, normal turn green, interrupt/recover orange, failed red, inactive gray.
- Visible Conversation header no longer carries a separate turn-state badge.
- Readiness summary no longer includes `Status`.
- Diagnostic panels remain available.
- Text turns update the rail through `REASON` and return to idle/final state.
- Resident PTT updates the rail through observed session states.
- Backend unavailable/startup failure appears as System State, not as a turn state.
- Desktop static tests protect the display contract.
