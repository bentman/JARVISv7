## Slice E — Local Service Substrate

**Date:** 2026-04-30
**Status:** Proposed
**Depends on:** Slice D closed and verified (SYSTEM_INVENTORY.md 2026-04-30)

---

## Purpose

Group E stands up the local service substrate — Redis and SearXNG — before any slice that consumes cache or search. Defining the infrastructure boundary here prevents each consumer from inventing its own connection logic. The backend continues to run without these services present. Every subsystem that needs them reports unavailable and degrades cleanly.

Settings flow from `.env` (runtime) and `.env.example` (documented template). No service URL, port, or API key is hardcoded in application code.

---

- Backend HTTP API established in D.1; consumed unchanged.
- `docker-compose.yml` at repo root already declares Redis and SearXNG with healthchecks and named volumes — the substrate definition exists; E.1 validates and corrects it.
- `config/cache/` directory exists with `.gitkeep`; it is reserved for application cache policy/config only (not Redis service/container/runtime configuration).
- `.env.example` exists; Group E extends it with service substrate entries.
- Regression baseline: 95 tests green on Windows x64 and Windows ARM64.

## Settings Flow Convention

All service URLs, ports, API keys, and enable/disable flags for Group E live in `.env` / `.env.example`. `docker-compose.yml` reads them via `${VAR}` substitution. Application code reads them via `backend/app/core/settings.py`. `.env` is gitignored; `.env.example` is the committed template.

Redis ownership for Slice E:
- Redis is configured primarily through `.env` / `.env.example` and `docker-compose.yml`.
- `cache/redis/` owns Redis Docker runtime cache/data mount contents.
- `config/redis/` is reserved for an explicit Redis config file only if a later sub-slice proves one is needed.

Search ownership for Slice E:
- `config/search/searxng/`, `config/search/searxng/cache/`, `config/search/ddgs/`, and `config/search/tavily/` own search service/provider configuration surfaces.
- `.env` / `.env.example` own search URLs, enable flags, and API-key placeholders.
- `config/models/search.yaml` is provider/routing config only and does not own credentials.

## Sub-Slice E.1 — Docker Service Substrate Verification

### Why Here

`docker-compose.yml` already exists but its mounted volume paths must align with the repo-owned config layout (`config/search/searxng/`, `cache/redis/`) and settings must flow from `.env`. E.1 corrects the compose file if needed and proves both services start and are reachable.

### Goal

`docker compose up` starts Redis and SearXNG; both reach healthy state; Redis responds to `PING`; SearXNG JSON search endpoint is reachable.

### Scope

**`docker-compose.yml`** — correct mount paths and env-var substitution:
- Redis data volume: `./cache/redis:/data`
- SearXNG config volume: `./config/search/searxng:/etc/searxng:rw`
- SearXNG cache volume: `./config/search/searxng/cache:/var/cache/searxng:rw`
- Redis port: `${REDIS_PORT:-6379}:6379`
- SearXNG port: `${SEARXNG_PORT:-8080}:8080`
- SearXNG base URL: `BASE_URL=${SEARXNG_BASE_URL:-http://localhost:8080}`
- Use `env_file: .env` at the compose service level so all vars flow from `.env`.

**`.env.example`** — extend with Group E entries:
```
# Local service substrate (Group E)
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=0

SEARXNG_BASE_URL=http://127.0.0.1:8080
SEARXNG_PORT=8080

# Search escalation
USE_SEARXNG=true
USE_DDGS=true
USE_TAVILY=false
TAVILY_API_KEY=
```

**`config/search/searxng/settings.yml`** — new; minimal SearXNG config:
- Preserve default settings behavior; only required change is adding `json` to `search.formats` so the JSON search API is available.
- Do not rewrite engine lists or broad defaults.
- Minimal shape:
  ```yaml
  server:
    base_url: ""
    bind_address: "0.0.0.0:8080"
  search:
    formats:
      - html
      - json
  ```

Docker prerequisite guidance remains in this slice document: Docker Desktop (or equivalent Docker runtime with `docker compose`) is required to run Group E substrate tests; `TAVILY_API_KEY` is optional.

### Files Not to Modify

- `backend/app/**` — no backend wiring in E.1
- `pyproject.toml` — no dependency changes in E.1

### Validation

```powershell

```


### Acceptance

```
PASS docker compose ps: redis health=healthy, searxng health=healthy
```

### Finish Line

## Sub-Slice E.5 — Search Wiring

### Why Here

Config is established (E.4) and services are proven (E.1). Search wiring lands last so Agent context is focused on wiring Jarvis to the configured search services, not on infrastructure or config concerns.

### Goal

A known query returns results via SearXNG when healthy; falls back to DDGS when SearXNG is stopped. Tavily path is present and policy-gated off by default.

### Scope

**`backend/app/runtimes/internetsearch/__init__.py`** — empty.

**`backend/app/runtimes/internetsearch/base.py`** — `SearchBase` ABC:
- `search(query: str, max_results: int = 5) -> list[SearchResult]`
- `is_available() -> bool`
- `SearchResult` dataclass: `title: str`, `url: str`, `snippet: str`, `source: str`

**`backend/app/runtimes/internetsearch/searxng_runtime.py`** — `SearXNGRuntime`:
- Uses `httpx`; reads `settings.searxng_base_url`.
- `is_available()`: GET `/healthz`; returns `False` with logged reason if unreachable.
- Returns `[]` on error without raising.

**`backend/app/runtimes/internetsearch/ddgs_runtime.py`** — `DDGSRuntime`:
- Uses `ddgs`; if an explicit import is needed, use `from ddgs import DDGS`; reads `settings.use_ddgs`.
- `is_available()`: returns `settings.use_ddgs`.
- Fail-closed on exception.

**`backend/app/runtimes/internetsearch/tavily_runtime.py`** — `TavilyRuntime`:
- Uses `httpx`; reads `settings.tavily_api_key` and `settings.use_tavily`.
- `is_available()`: returns `False` if `use_tavily` is False or `tavily_api_key` is empty.

**`backend/app/routing/runtime_selector.py`** — extend with `select_search_runtime(settings) -> SearchBase`:
- Prefers SearXNGRuntime → DDGSRuntime → TavilyRuntime per settings flags.
- Returns `NullSearchRuntime` (returns `[]`, logs reason) if no provider is available.

**`backend/tests/unit/runtimes/internetsearch/test_search_runtime.py`**:
- `test_searxng_is_available_false_when_unreachable` (mock httpx)
- `test_searxng_returns_results_on_valid_response` (mock httpx)
