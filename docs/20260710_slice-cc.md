# Slice CC - Performance Optimization
Status: planned. This document is an implementation contract for a coding agent. It is not completion evidence.

---

## Purpose
Optimize system performance by eliminating unnecessary overhead in high-frequency, latency-critical paths. This slice addresses network connection churn in the local LLM reasoning and polling infrastructure, and reduces Time-to-First-Audio (TTFA) by introducing streaming partial playback to the TTS runtime layer.

---

## Agent Contract
- Preserve existing validated behavior unless this slice explicitly changes it.
- Keep scope limited to the files and behavior named in this slice.
- Search runtimes are explicitly excluded from this slice.

---

## Baseline Consumed
- `backend/app/runtimes/llm/local_runtime.py` provides local LLM inference but uses module-level `httpx.post`.
- `backend/app/runtimes/llm/ollama_runtime.py` provides Ollama LLM inference but uses module-level `httpx.post`.
- Polling services (`local_llm_startup.py`, `local_llm_sidecar.py`, `service_status.py`) use module-level `httpx.get`.
- `backend/app/runtimes/tts/base.py` defines a synchronous `synthesize` method returning a single `np.ndarray`.
- `backend/app/runtimes/tts/kokoro_onnx_runtime.py` synthesizes entire audio responses before returning.
- `backend/app/runtimes/tts/playback.py` plays audio synchronously via `sounddevice.play`.
- `backend/app/conversation/engine.py` orchestrates voice turns and blocks on TTS synthesis before beginning playback.
- `backend/app/runtimes/stt/onnx_whisper_runtime.py` explicitly checks the filesystem for model availability on every transcription call.
- `backend/app/services/utterance_segmenter.py` appends pure trailing silence to the STT audio payload, causing unnecessary encoder overhead.

---

## Out of Scope
- No changes to search runtimes (SearXNG, DDGS, Tavily).
- No structural changes to the synchronous/asynchronous design of the FastAPI application.
- No modifications to `TurnEngine` beyond the TTS synthesis and playback flow.

---

## Target Observable Behavior
- LLM inference latency is reduced because TCP handshakes are no longer performed for each request.
- TTS Time-to-First-Audio (TTFA) is dramatically reduced as the assistant begins speaking the first sentence while subsequent sentences synthesize in the background.
- STT Time-To-First-Token (TTFT) and overall inference speed are improved by removing redundant disk I/O and pruning silent audio payloads, which also reduces hallucination rates.
- No direct UI surface changes; behavior is observable through decreased latency and smoother conversational pacing.

## CC.1 - HTTP Connection Pooling

### Goal
Refactor the LLM runtimes to instantiate and reuse a persistent `httpx.Client` on initialization, and wrap polling services in short-lived `httpx.Client` context managers to share the connection pool.

### Scope
Create/modify:
- `backend/app/runtimes/llm/local_runtime.py` — instantiate `httpx.Client` in `__init__` and use for API calls.
- `backend/app/runtimes/llm/ollama_runtime.py` — instantiate `httpx.Client` in `__init__` and use for API calls.
- `backend/app/services/local_llm_startup.py` — use `with httpx.Client() as client` for polling loop.
- `backend/app/services/local_llm_sidecar.py` — use `with httpx.Client() as client` for polling loop.
- `backend/app/api/service_status.py` — use `httpx.Client()` context blocks for health probes.

### Files Not to Modify
- `backend/app/runtimes/internetsearch/*.py` — Out of scope.

### Validation
1. `pytest backend/tests` (focused on LLM runtimes or HTTP clients).
2. Desktop App: Run `npm --prefix desktop test` and verify the UI reasoning states are not disturbed.
3. Add a `CHANGE_LOG.md` entry for these code changes.

### Acceptance
```text
PASS for unit and integration validation.
LLM runtimes use instantiated httpx.Client over module-level functions.
```

## CC.2 - TTS Runtime Layer

### Goal
Implement streaming partial assistant response in the TTS layer to dramatically reduce TTFA (Time-to-First-Audio), by streaming synthesized audio chunks directly into the playback buffer.

### Scope
Create/modify:
- `backend/app/runtimes/tts/base.py` — Add an abstract method `synthesize_stream(self, text: str) -> Iterable[np.ndarray]`.
- `backend/app/runtimes/tts/kokoro_onnx_runtime.py` — Implement `synthesize_stream` using the underlying `kokoro.create_stream` generator.
- `backend/app/runtimes/tts/playback.py` — Add a `play_stream` function that accepts an iterable of audio chunks and writes them to a `sounddevice.OutputStream`.
- `backend/app/conversation/engine.py` — Modify `_speak_or_degrade` and `_play_with_interruption_monitor` to consume `synthesize_stream` and pipe to `play_stream`, while maintaining barge-in detection on the incoming chunks.

### Files Not to Modify
- Non-TTS runtimes.

### Validation
1. `pytest backend/tests` (focused on TTS and `engine.py`).
2. Desktop App: Run `npm --prefix desktop test` and verify voice playback visual indicators are not disturbed.
3. Add a `CHANGE_LOG.md` entry for these code changes.

### Acceptance
```text
PASS for unit and integration validation.
TTS output plays incrementally before full synthesis completes.
```

## CC.3 - STT Surface Layer

### Goal
Eliminate unnecessary overhead in the STT path by stripping redundant disk I/O on the transcription hot-path and precisely trimming trailing silence from audio buffers to reduce STT encoder workload and hallucination probability.

### Scope
Create/modify:
- `backend/app/runtimes/stt/onnx_whisper_runtime.py` — Remove `self.is_available()` check from `transcribe()` in both runtime classes, as files are already verified during warmup.
- `backend/app/services/utterance_segmenter.py` — Precisely slice off the `silence_after_speech` tail from the returned `UtteranceSegment` audio payload so STT avoids processing empty Mel frames.

### Files Not to Modify
- Non-STT and non-segmenter service logic.

### Validation
1. `pytest backend/tests` (focused on `utterance_segmenter.py` and STT runtimes).
2. Desktop App: Run `npm --prefix desktop test` and verify recording and voice input bounds are not disturbed.
3. Add a `CHANGE_LOG.md` entry for these code changes.

### Acceptance
```text
PASS for unit and integration validation.
STT transcribes audio without initiating synchronous disk I/O for model files.
Audio payload size is strictly bounded by active speech bounds without trailing silence.
```

## CC.4 - Conversation Engine Surface

### Goal
Fix a structural accuracy flaw where LLM-generated code blocks and markdown are permanently deleted from system memory because the engine unconditionally applies TTS sanitization (even for text turns) and records the sanitized output to episodic memory. Optimize the hot-path to eliminate redundant policy compilations.

### Scope
Create/modify:
- `backend/app/conversation/engine.py` — Update `_run_reasoning_path` to preserve the original `response` in `TurnResult.response_text` so memory accurately reflects generated code/markdown. Derive `tts_text` explicitly and pass it to `_speak_or_degrade` for synthesis.
- `backend/app/cognition/prompt_assembler.py` — Modify `assemble_prompt_envelope` to accept `policy: PersonalityPolicy` directly, preventing redundant policy compilations per turn.

### Files Not to Modify
- Memory storage implementations (`backend/app/memory/*.py`).

### Validation
1. `pytest backend/tests/` (focused on `engine.py`) to assert `TurnResult.response_text` is unadulterated.
2. Desktop App: Run `npm --prefix desktop test` and verify that generated code blocks and markdown are correctly rendered in the chat UI.
3. Add a `CHANGE_LOG.md` entry for these code changes.

### Acceptance
```text
PASS for unit and integration validation.
Episodic memory records retain full code block and markdown formatting (unaffected by TTS sanitization).
```
