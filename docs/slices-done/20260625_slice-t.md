# 20260625_slice-t.md
## Slice T - Resident Full-Duplex Voice Layer

---

## Purpose

Slice T adds a thin resident full-duplex voice layer around the existing Python backend, realtime session, and `TurnEngine` path. The objective is to replace fixed-window resident voice capture with one shared microphone stream, evidence-backed utterance boundaries, wake/PTT handoff through the same audio path, live interruption detection during TTS, and explicit hands-free modes. This slice exists because `ProjectVision.md` requires voice-first invocation, natural interruption, visible states, and recoverable failures, while current repo evidence shows resident PTT and wake are still batch/fixed-window capture paths.

Done means resident voice can be validated as a backend-owned, desktop-visible interaction layer that preserves the canonical turn engine, artifacts, memory, tools, personality, local runtime selection, and fallback PTT behavior.

---

## Baseline Consumed

- `ProjectVision.md`: voice-first, local-first, interruptible assistant target; consumed unchanged as the product contract.
- `SYSTEM_INVENTORY.md`: Slice B voice runtimes, Slice C turn/session engine, Slice D/J/K desktop surface, Group M realtime boundary, Group R/S local LLM runtime state; consumed as current capability truth.
- `backend/app/services/voice_service.py`: current `sounddevice` blocking capture and wake chunk source; extended, not replaced wholesale.
- `backend/app/services/wake_monitor.py`: current openWakeWord monitor with pre-roll and fixed post-wake capture; extended to consume a shared stream.
- `backend/app/services/resident_voice_invocation.py`: current resident PTT/wake invocation queue; extended as the only enqueue path for PTT, wake, hands-free, and barge-in turns.
- `backend/app/conversation/realtime/session.py`: current realtime event boundary and committed voice invocation path; extended without replacing `TurnEngine`.
- `backend/app/conversation/engine.py`: canonical text/voice turn engine and interruption seam; consumed unchanged until live interruption chunks are available.
- `backend/app/runtimes/stt/barge_in.py`: current RMS guard-time detector; extended or wrapped, not treated as final acoustic barge-in.
- `backend/app/runtimes/tts/playback.py`: current non-blocking `start()` plus blocking `play()` and `stop()` functions; extended through orchestration, not duplicated.
- `backend/app/api/routes/session.py` and `backend/app/api/schemas/session.py`: current `/session/ptt` and status surface; extended for modes only after backend behavior exists.
- `desktop/src/main.js` and `desktop/src/components/resident-voice.js`: current PTT/status UX; extended after backend state exists.
- `pyproject.toml`: declares `sounddevice` in base dependencies; `openwakeword` and ONNX Runtime enter through hardware extras selected by provisioning. Any VAD package or model acquisition change must go through the same provisioning rules.

## Out of Scope

- No replacement of `TurnEngine`, realtime session, session manager, artifact storage, memory, tool execution, personality, or LLM runtime selection.
- No browser-first microphone architecture, `getUserMedia`, AudioWorklet, WebSocket audio transport, or duplicated desktop audio path in this slice.
- No cloud STT/TTS, remote wake detection, or cloud-based interruption logic.
- No semantic memory, autonomous agents, planner/executor activation, or model-side tool calling.
- No new hardware host-detection logic outside `backend/app/hardware/profiler.py` and no DLL/path bootstrap outside `backend/app/hardware/preflight.py`.
- No hand edits to `backend/requirements.txt`.
- No always-on hands-free default; hands-free and accessibility continuous listen must be explicit modes.
- No claim of echo cancellation, overlapped speech transcription, or perfect acoustic barge-in unless live validation proves it.

## Slice-Level Closeout Criteria

1. Resident PTT uses VAD-delimited streamed capture when available and preserves blocking fallback when unavailable.
2. Wake and PTT no longer compete for separate microphone streams.
3. Wake command capture uses utterance end detection instead of a fixed three-second post-wake window.
4. No-speech after wake and microphone/VAD failures produce explicit visible degraded or failed states.
5. Assistant speech can be interrupted through live microphone evidence and records interruption metadata.
6. Accepted barge-in can enqueue a follow-up voice turn with `source="barge_in"`.
7. Desktop exposes resident voice modes without making hands-free the default.
8. `backend/.venv/Scripts/python scripts/validate_backend.py unit` passes on the current host.
9. `backend/.venv/Scripts/python scripts/validate_backend.py regression` passes on the current host.
10. Live resident voice evidence is recorded for each host class claimed; untested host classes remain explicitly unclaimed or `SKIP-<reason>`.

## Risks / Sequencing Notes

| Risk | Severity | Mitigation |

| VAD dependency or model artifact is not currently declared | High | Resolve the T.2 dependency decision before implementation; prefer existing ONNX Runtime or dependency-free fallback unless a provisioned dependency is approved. |
