# 20260425-slice_b.md
## Slice B — Cross-Platform Voice Runtime Strategy

**Date:** 2026-04-25
**Status:** Proposed
**Depends on:** Slice A closed and verified (SYSTEM_INVENTORY.md 2026-04-24 17:16)

---

## Purpose

Introduce one runtime family per voice subsystem (STT, TTS, LLM, Wake), each accepting `device` as a constructor parameter. Device selection is driven exclusively by Slice A readiness evidence (`derive_*_device_readiness()` outputs). CPU paths proven on both Windows host classes. Acceleration branches proven where hardware is present. B.5 acceleration matrix established as a hard gate before Group C begins.

---

## Slice A Baseline Consumed

All of the following are verified in `SYSTEM_INVENTORY.md` and consumed unchanged:

- `run_profiler()` → `FullCapabilityReport` including `CapabilityFlags` (`qnn_available`, `directml_candidate`, `supports_cuda_llm`)
- `run_preflight()` → `PreflightResult` with evidence tokens (`import:*`, `ep:*`, `dll:*`)
- `derive_{stt,tts,llm,wake}_device_readiness(preflight, profile)` → `(device, ready, reason)` tuples — the sole input to runtime selectors; runtimes never re-derive hardware policy
- `resolve_required_extras()` and provisioned extras: `onnxruntime`, `onnx-asr`, `kokoro-onnx`, `openwakeword` are already installed on provisioned hosts via `hw-x64-base` / `hw-arm64-base`; `pvporcupine>=3.0` is in base
- `scripts/validate_backend.py` with `runtime`, `matrix`, and `regression` subcommands operational
- `scripts/ensure_models.py` stub exists; activated for model families in this slice
- `conftest.py` `SKIP_UNLESS_*` constants and `SKIP_UNLESS_LIVE` env-var gate operational

## Out of Scope

Slice B introduces no Group C/D/E/F/G/H/I implementation scope. Specific exclusions:

- No conversation engine, turn states, or session management
- No backend API routes or services layer
- No desktop shell code
- No Redis, SearXNG, or cache layer
- No tools, memory, or agent framework files

## Pre-Implementation Gates

The following must be resolved before Agent begins implementation. Each is an explicit verification step, not an assumption.

**Gate B-G1 — ONNX Whisper `small` HF file layout.**
The pre-exported ONNX repo is `onnx-community/whisper-small` on HuggingFace. The export is split (not monolithic): `encoder_model.onnx` + `decoder_model_merged.onnx` in an `onnx/` subfolder, alongside `config.json` and tokenizer files. Agent must confirm the exact filenames from the HF file tree before writing the inference session call and the `stt.yaml` catalog entry — do not hardcode.

**Gate B-G2 — Kokoro-onnx model source.**
The `kokoro-onnx` package uses two files: `kokoro-v1.0.onnx` (310 MB f32) and `voices-v1.0.bin`. These are released at `https://github.com/thewh1teagle/kokoro-onnx/releases` — not from HuggingFace Hub directly. `ensure_models.py --family tts` should download both files from the GitHub release URLs for the version pinned in `tts.yaml`. Agent must confirm the exact release tag and URLs before writing the catalog entry. Note: a lighter `kokoro-v1.0.int8.onnx` (88 MB) is also available from the same release and may be preferred for ARM64 — confirm suitability.

**Gate B-G3 — openWakeWord Windows embedding model file.**
Resolved: the `dscripka/openWakeWord` v0.5.1 release provides `hey_jarvis_v0.1.onnx`, `melspectrogram.onnx`, and `embedding_model.onnx` — all `.onnx`. No `.tflite` file is required on Windows. The `wake.yaml` catalog entries use the confirmed `.onnx` URLs. No further verification needed before Agent writes the catalog.

**Gate B-G4 — `huggingface_hub` dependency.**
`ensure_models.py` requires `huggingface_hub` for HF artifact downloads. This package is not currently in `pyproject.toml`. Decision: add to base `[project].dependencies` (recommended — model acquisition is required on every host) or `dev` extra. This requires a `pyproject.toml` edit and re-provisioning before B.1 model acquisition can run. Resolve before Agent begins B.1.

**Gate B-G5 — `SKIP_UNLESS_OLLAMA` sentinel.**
Current `conftest.py` `SKIP_UNLESS_OLLAMA` uses `importlib_util.find_spec("ollama")` which checks for the `ollama` Python package — not whether the Ollama server is running. The `ollama` Python package is not in any current extra. Decision: replace with an env-var sentinel (`JARVISV7_OLLAMA_URL` set = server present, consistent with `SKIP_UNLESS_LIVE` pattern), or add a lightweight HTTP probe. Resolve before Agent begins B.3. The `ollama` Python package itself is not required — `httpx` is sufficient for the Ollama HTTP API.

**Gate B-G6 — Porcupine test marker.**
B.4 live Porcupine tests require a `PICOVOICE_ACCESS_KEY` gate. Decision: add `SKIP_UNLESS_PORCUPINE = os.getenv("PICOVOICE_ACCESS_KEY") is None` to `conftest.py` and add a `requires_porcupine` marker to `pyproject.toml` `[tool.pytest.ini_options].markers`. Agent adds both as part of B.4.
