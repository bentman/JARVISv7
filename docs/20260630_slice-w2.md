## Slice W2 — Settings Grouping and Default Correction

Status: planned. This document is an implementation contract for a coding agent. It is not completion evidence.

---

## Purpose

Correct the parts of Slice W that landed incorrectly. Slice W2 is not a continuation that assumes the Slice W implementation is the new source of truth. It is a correction contract that compares Slice W output against the starter environment posture, backend defaults, Operator Settings grouping, and desktop static contracts.

If Slice W behavior conflicts with this document, implement the W2 correction rather than preserving the Slice W behavior.

---

## Correction Targets

1. The starter env file has a direct typo: the tool sandbox path and config path were concatenated into one line. Split them into separate entries and cover this with an existing env parsing test.
2. Decide whether the LLM model-root env name is a real supported setting. If required, add a Settings field with tests. If not supported, remove it from W2 required defaults.
3. Backend defaults must match the intended starter posture: local llama.cpp enabled, model policy auto, Docker search disabled unless enabled, Ollama disabled but with development defaults available, Ollama model phi4-mini, Ollama context 8192, and Ollama localhost port 11434.
4. Operator grouping must split local llama.cpp from local Ollama app. Implement this in backend metadata; keep desktop rendering metadata-driven unless it cannot display the corrected sections.
5. Do not update changelog or inventory during this correction pass.

## Agent Contract

- Preserve supported environment names unless a tested compatibility path replaces them.
