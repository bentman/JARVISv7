## Slice B тАФ Cross-Platform Voice Runtime Strategy

**Date:** 2026-04-25
**Status:** Proposed

## Purpose

Introduce one runtime family per voice subsystem (STT, TTS, LLM, Wake), each accepting `device` as a constructor parameter. Device selection is driven exclusively by Slice A readiness evidence (`derive_*_device_readiness()` outputs). CPU paths proven on both Windows host classes. Acceleration branches proven where hardware is present. B.5 acceleration matrix established as a hard gate before Group C begins.

---

## Slice A Baseline Consumed

All of the following are verified in `SYSTEM_INVENTORY.md` and consumed unchanged:

- `run_profiler()` тЖТ `FullCapabilityReport` including `CapabilityFlags` (`qnn_available`, `directml_candidate`, `supports_cuda_llm`)
- `run_preflight()` тЖТ `PreflightResult` with evidence tokens (`import:*`, `ep:*`, `dll:*`)
- `derive_{stt,tts,llm,wake}_device_readiness(preflight, profile)` тЖТ `(device, ready, reason)` tuples тАФ the sole input to runtime selectors; runtimes never re-derive hardware policy
- `resolve_required_extras()` and provisioned extras: `onnxruntime`, `onnx-asr`, `kokoro-onnx`, `openwakeword` are already installed on provisioned hosts via `hw-x64-base` / `hw-arm64-base`; `pvporcupine>=3.0` is in base
- `scripts/validate_backend.py` with `runtime`, `matrix`, and `regression` subcommands operational
- `scripts/ensure_models.py` stub exists; activated for model families in this slice
- `conftest.py` `SKIP_UNLESS_*` constants and `SKIP_UNLESS_LIVE` env-var gate operational

## Pre-Implementation Gates

The following must be resolved before Agent begins implementation. Each is an explicit verification step, not an assumption.

**Gate B-G1 тАФ ONNX Whisper `small` HF file layout.**
Inspect `openai/whisper-small` on HuggingFace. Determine: (a) exact subfolder and filenames (monolithic `model.onnx` vs split `encoder_model.onnx` + `decoder_model.onnx`), (b) whether a pre-exported ONNX variant is available or whether export is required. The `config/models/stt.yaml` catalog entry and `onnx_whisper_runtime.py` session creation must match the actual file layout. Agent must not hardcode filenames before this is verified.

**Gate B-G2 тАФ Kokoro-onnx model source.**
Verify: (a) the HuggingFace repo ID for the Kokoro ONNX model artifacts (candidate: `hexgrad/Kokoro-82M`), (b) exact filenames for the model pair, (c) whether `kokoro-onnx` exposes a built-in download helper or whether `ensure_models.py` must call `huggingface_hub` directly. The `config/models/tts.yaml` entry and `ensure_models.py --family tts` depend on this.

**Gate B-G3 тАФ openWakeWord Windows embedding model file.**
The v0.5.1 release contains both `embedding_model.onnx` and `embedding_model.tflite`. On Windows, `tflite-runtime` is unavailable, so the `.onnx` variant must be used. Verify that `openwakeword.Model()` on Windows correctly loads the `.onnx` backend when given the `.onnx` embedding file. `config/models/wake.yaml` must reference the correct filename.

**Gate B-G4 тАФ `huggingface_hub` dependency.**
`ensure_models.py` requires `huggingface_hub` for HF artifact downloads. This package is not currently in `pyproject.toml`. Decision: add to base `[project].dependencies` (recommended тАФ model acquisition is required on every host) or `dev` extra. This requires a `pyproject.toml` edit and re-provisioning before B.1 model acquisition can run. Resolve before Agent begins B.1.

**Gate B-G5 тАФ `SKIP_UNLESS_OLLAMA` sentinel.**
Current `conftest.py` `SKIP_UNLESS_OLLAMA` uses `importlib_util.find_spec("ollama")` which checks for the `ollama` Python package тАФ not whether the Ollama server is running. The `ollama` Python package is not in any current extra. Decision: replace with an env-var sentinel (`JARVISV7_OLLAMA_URL` set = server present, consistent with `SKIP_UNLESS_LIVE` pattern), or add a lightweight HTTP probe. Resolve before Agent begins B.3. The `ollama` Python package itself is not required тАФ `httpx` is sufficient for the Ollama HTTP API.

**Gate B-G6 тАФ Porcupine test marker.**
B.4 live Porcupine tests require a `PICOVOICE_ACCESS_KEY` gate. Decision: add `SKIP_UNLESS_PORCUPINE = os.getenv("PICOVOICE_ACCESS_KEY") is None` to `conftest.py` and add a `requires_porcupine` marker to `pyproject.toml` `[tool.pytest.ini_options].markers`. Agent adds both as part of B.4.

|--------|-----|------|----------|-----|
