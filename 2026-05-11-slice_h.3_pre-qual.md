**Date:** 2026-05-11

## Purpose

H.3 failed previously because the proof chain was never completed: `QNNExecutionProvider` session initialization was proven (H.2), but the critical step — a known-audio fixture producing a correct transcript through a QNN-accelerated Whisper path — was not.

The root cause was an artifact/runtime contract mismatch. The `whisper-tiny-qnn-precompiled-snapdragon-x-elite` artifact did not satisfy the `QnnWhisperRuntime` encode→decode contract, and the transcription loop was never implemented. H.3 was deliberately reverted and marked `Deferred` at that point.

This pre-qual task resolves that root cause before any wiring code is written. Its single output is a binary decision: **PROCEED** (artifact and runtime contract proven) or **DEFER** (named stop condition documented).

---

## Why AI Hub Models is the Correct Acquisition Path

Qualcomm AI Hub Models (`qai_hub_models`) is Qualcomm's first-party PyPI-published SDK that compiles, profiles, and downloads pre-optimized model artifacts from Qualcomm AI Hub Workbench. For Whisper specifically:

- `whisper_base_en` is a published AI Hub model with Qualcomm-owned HuggingFace assets at `qualcomm/Whisper-Base-En`.
- The export script supports `--target-runtime precompiled_qnn_onnx` targeting the `Snapdragon X Elite CRD` device.
- The resulting artifact is a `precompiled_qnn_onnx` pair: `WhisperEncoder.onnx` + `WhisperDecoder.onnx` each containing an embedded QNN context binary runnable via `QNNExecutionProvider` on Windows ARM64.
- This is the same artifact format used in `FluidInference/whisper-large-v3-turbo-qnn` and in the Qualcomm AI Hub compile documentation for Windows X Elite targets.

Key constraint: `qai_hub_models` requires **x64 Python** for export/compilation jobs even on a Snapdragon X Elite machine. The downloaded artifact files are OS-agnostic and are deployed using ARM64 Python + `onnxruntime-qnn` for inference. This is documented explicitly in the AI Hub Models README: `"Only AMDx64 (64-bit) Python is supported on Windows. Installation will fail when using Windows ARM64 Python."`

The AI Hub Workbench orchestrates compilation on a cloud-hosted Snapdragon X Elite CRD device, so no local QAIRT toolchain is required for the export step.

Record the full input/output tensor contract for both sessions verbatim. This is mandatory — H.3 implementation depends on these exact names and shapes. Typical AI Hub whisper-base-en contract:
- Encoder input: `input_features` shape `(1, 80, 3000)` float16 or float32
- Encoder output: `audio_embed` or `hidden_states` shape `(1, 1500, 512)`
- Decoder inputs: `decoder_input_ids`, `encoder_hidden_states`, plus per-layer KV-cache tensors (names and shapes vary by compilation; record all of them)
- Decoder output: `logits` shape `(1, N, 51865)` where N is the input sequence length

## Step 7 — Live Transcript Proof using the Known-Audio Fixture

This is the decisive gate. Using `backend/tests/fixtures/hello_world.wav` (expected transcript: `"hello world"`), attempt a complete encode→decode inference pass.

Run this as a standalone probe script (not committed to repo):

```powershell
backend\.venv\Scripts\python -c "
import wave, numpy as np
from pathlib import Path
from backend.app.hardware.qnn_provider import create_qnn_session

# Load known-audio fixture
wav_path = Path('backend/tests/fixtures/hello_world.wav')
with wave.open(str(wav_path), 'rb') as wf:
    frames = wf.readframes(wf.getnframes())
    sample_rate = wf.getframerate()
    n_channels = wf.getnchannels()
audio = np.frombuffer(frames, dtype=np.int16).astype(np.float32) / 32768.0
if n_channels > 1:
    audio = audio[::n_channels]
print(f'audio: shape={audio.shape}  sr={sample_rate}')

# Mel spectrogram extraction — install transformers temporarily if needed:
# backend\.venv\Scripts\pip install transformers
from transformers import WhisperFeatureExtractor
feat = WhisperFeatureExtractor.from_pretrained('openai/whisper-base')
mel = feat(audio, sampling_rate=sample_rate, return_tensors='np')['input_features']
print(f'mel: shape={mel.shape}  dtype={mel.dtype}')

# Encoder pass — adjust input name from Step 6 output if different from 'input_features'
enc_path = Path(r'C:\tmp\whisper_base_en_qnn_x_elite\WhisperEncoder.onnx')
enc_session, enc_method = create_qnn_session(enc_path, disable_cpu_fallback=True)
print(f'encoder providers: {enc_session.get_providers()}  method: {enc_method}')
enc_outputs = enc_session.run(None, {'input_features': mel})
hidden_states = enc_outputs[0]
print(f'encoder hidden_states: shape={hidden_states.shape}  dtype={hidden_states.dtype}')

# Decoder single-step proof — adjust input names from Step 6 output
# NOTE: if decoder requires KV-cache inputs, initialize as zeros with shapes from Step 6
dec_path = Path(r'C:\tmp\whisper_base_en_qnn_x_elite\WhisperDecoder.onnx')
dec_session, dec_method = create_qnn_session(dec_path, disable_cpu_fallback=True)
print(f'decoder providers: {dec_session.get_providers()}  method: {dec_method}')

SOT_ID = 50258  # Whisper-base-en start-of-transcript token
decoder_input_ids = np.array([[SOT_ID]], dtype=np.int32)

# Build decoder feed_dict — add KV-cache inputs as zeros if required by contract from Step 6
feed = {
    'decoder_input_ids': decoder_input_ids,
    'encoder_hidden_states': hidden_states,
    # add KV-cache inputs here if Step 6 showed them, e.g.:
    # 'past_key_values.0.decoder.key': np.zeros((1, 8, 0, 64), dtype=np.float32),
}
dec_outputs = dec_session.run(None, feed)
logits = dec_outputs[0]
next_token = int(np.argmax(logits[0, -1, :]))
print(f'decoder logits shape: {logits.shape}')
print(f'first predicted token id: {next_token}')
print('DECODER SINGLE STEP: PASS')
print('H.3 PRE-QUAL STEP 7: COMPLETE — full greedy decode loop is H.3 implementation scope')
"
```

The decisive pass conditions for Step 7:
1. Mel spectrogram produced without error
2. Encoder runs and produces a `hidden_states` tensor with a plausible shape (e.g., `(1, 1500, 512)`)
3. Decoder runs one step without error and produces a logits tensor with a plausible shape (e.g., `(1, 1, 51865)`)
4. `QNNExecutionProvider` is confirmed primary for both encoder and decoder

A correct end-to-end transcript is **not** required here — the full greedy decode loop is H.3 implementation scope. Mechanical soundness of the inference path is what this step proves.

| ARM64 regression drops below 102 | Stop immediately | Do not proceed; isolate cause before continuing |

## Notes

- The AI Hub export step requires internet access and a Qualcomm ID. If the ARM64 host has no internet access, run Steps 1–3 on any x64 Windows machine and transfer the `.onnx` files to the ARM64 host for Steps 5–8.
- `whisper-base-en` (74M parameters) is recommended over `whisper-tiny-en` (39M) as it matches the existing CPU model size family more closely. If base export fails, `whisper_tiny_en` is the named fallback — document the substitution explicitly and update the target catalog ID in H.3 accordingly.
- The `transformers` library in Step 7 may not be in `backend/.venv`. If not: `backend\.venv\Scripts\pip install transformers` temporarily for the probe. Do not add `transformers` to `pyproject.toml` as part of this pre-qual.
- The `QnnWhisperRuntime` in `onnx_whisper_runtime.py` uses `create_qnn_session` from `qnn_provider.py` and currently expects `encoder.onnx` / `decoder.onnx`. The AI Hub export may produce `WhisperEncoder.onnx` / `WhisperDecoder.onnx`. Document the actual names in Step 4 — H.3 wiring must reference the exact filenames.
- KV-cache handling is the most likely implementation complexity in H.3. The AI Hub Whisper decoder is compiled with fixed KV-cache inputs (static shapes per attention head × layers). Record these shapes precisely in Step 6; the H.3 decode loop must initialize them correctly or the decoder will fail with a shape mismatch.
